# 温湿度連動型・自律ロボットペット開発プロジェクト

- 本リポジトリは、温湿度センサーやシリアル通信と連動する **「卓上型の自律型ロボットペット」**の設計・組み込みソースコードを管理しています。
- 「電源を挿すだけで自律的に振る舞う」をコンセプトにした、Arduino UNO R4 WiFiベースの卓上型インタラクティブ・ガジェットです。
- 無機質な置き時計に、独自のロジックとマルチタスク制御で「生きてる感」を吹き込みました。

## プロジェクトの目的

- **組み込みスキルの可視化**: FreeRTOSを用いたタスク制御、メモリ制限下での幾何学描画など、低レイヤの技術力を統合的に証明する。
- **表現力の追求**: LOVOTのような「愛着の湧くインターフェース」を、数式による描画ロジックで実現する。

## 技術的な見どころ

### 1. FreeRTOS によるマルチタスクアーキテクチャ

「絶対にフリーズしないUI」を実現するため、処理を3つのタスクに完全分担させました。

- **タスクA（アニメーション）**: 目の描画・視線移動（低優先度）
- **タスクB（UI・ボタン監視）**: 10ms周期でボタンを監視し、即座に時刻モードへ切替（高優先度）
- **タスクC（天気通信）**: Wi-Fi経由でOpenWeatherMap APIと通信（中優先度）
  - **メリット**: 通信処理や複雑な描画計算を行っている間も、ボタン入力が全く途切れない堅牢なUIを実現しています。

### 2. 数式による「モーフィング」描画

ビットマップデータを一切使わず、`fillEllipse` 等の幾何学関数と数式で表情を生成。

- **メリット**: SRAM消費を最小限に抑えつつ、状態遷移時にフレーム補間（lerp）をかけることで、デジタルなのに「ぬるっ」とした人間味のある表情変化を実装しました。

### 3. 非同期タイマーによる生命感の演出

`delay()` を一切排除し、すべて `millis()` を用いた状態遷移で管理。

- **まばたき**: 3〜7秒間隔でランダムに発生。
- **自律ステート管理**: `enum` による感情管理と、確率の重み付け（NORMAL 55%, HAPPY 25%等）により、単調ではない動きを実現。

## 動作仕様

- **アニメーションモード**: 感情に応じた目・視線の自律移動
- **天気連動**: 気温・天気コードに応じて、「暑い」「悲しい（雨）」等の感情が自動でモーフィング変化。
- **時刻表示モード**: ボタン操作で即座に切り替え、5秒後に自動復帰。
