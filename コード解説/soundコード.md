# sound.cpp

- キャラクターの喜怒哀楽に合わせて、「たまごっち」や「ピカチュウ」のような**可愛いピコピコ音（電子音）を鳴らす「声帯・オーディオエンジン」の役割を持つコード**
- Arduino UNO R4に搭載されているアナログ出力機能（DAC）を最大限に活かし、ただ単に音を鳴らすだけでなく、耳に優しい音色へのチューニングや、電子工作で発生しやすい「ジー」というノイズの徹底的な除去を行なっている

## 1. 音作りの心臓部：`r4_dac_tone`（電子音生成）

- この関数は、指定された周波数（音の高さ）と長さ（ミリ秒）をもとに、実際に電気の波（矩形波）を作成

### 耳に優しい音色の秘密（音量・音割れ対策）

```cpp
#define DAC_VOLUME_LEVEL  12

```

- 通常、Arduinoで最大の電気を流すとスピーカーの性能限界を超えて「バリバリ」と不快な音割れを起こしてしまうため、このコードでは、出力レベルを `12`（最大255段階中）というかなり低い値に抑えてる
- これにより、電圧のトゲが丸くなり、**レトロゲーム機や平成の電子ペットのような、ポコポコとした角の丸い優しい音色**を作り出している

### ノイズ完全抹殺処理（最重要テクニック）

- 電子工作スピーカーの最大の敵でもある、音が鳴っていない「無音時」にアンプが電気の乱れを拾って「ジジジ…」と鳴り続けるノイズを防ぐため、このコードでは、音が鳴り終わった直後に以下の強力な対策を行っている

```cpp
analogWrite(DAC_PIN, 0);   // ① 電圧を完全に0にする
pinMode(DAC_PIN, INPUT);   // ② ピンを「INPUT」にして電気の通り道を物理的に遮断
```

- 音を出力するピンを、あえて「入力を受け付ける状態（INPUT）」に変えることで、スピーカーへ繋がる電気の回路を疑似的に遮断し、**音が止まった瞬間に完全な静寂（ノイズゼロ）を作成**

## 2. 感情ごとの「鳴き声」デザイン

- `playCry(emotion)` 関数の中では、画面の表情アニメーションと完全に同期する形で、7つの感情に合わせたメロディがドット絵のように細かく定義されている

- **`EMOTION_NORMAL` (通常) [ピカチュウ風]**
  - 「C6 → E6 → G6 → A6 → F6 → D6」と、音階を細かく階段状に上下させることで、生き物が本当におしゃべりしているかのような抑揚（イントネーション）を再現

- **`EMOTION_HAPPY` (嬉しい) / `EMOTION_BATH` (お風呂) [たまごっち風]**
  - 明るい和音（ド・ミ・ソ・ド・ミ・ソ）を **30ms（100分の3秒）という超高速スピード**で駆け上がらせることで、ゲームのレベルアップ音のような「ピロリロリン♪」というご機嫌な効果音を作成

- **`EMOTION_SAD` (悲しい) [メソメソ・ぴえん]**
  - 音階を「ド → シ → シ♭ → ラ」と、半音ずつ滑り落ちるように低くしている
  - 音楽理論でいう「短調（マイナー）」の響きが強調され、トーンがしょんぼり聞こえる工夫

- **`EMOTION_ANGRY` (怒り) [トゲトゲ効果]**
  - 音と音の間に「10ms」の鋭い無音（隙間）を強制的に挟むことで、音が「プッ！プッ！」と尖り、怒って頬をプンプン膨らませているような苛立ちを表現

- **`EMOTION_SLEEPY` (うとうと) [あくびの表現]**
  - 「ふぁ〜〜」とあくびをするように1つめの音を300msと長く伸ばし、途中で `delay(80)` という「一瞬息が止まるタメ」を作ってから、最後にポツンと低い音を鳴らして消えるという、ストーリー性のある贅沢な構成になっている

- **`EMOTION_FEAST` (ごちそう) [お祭りファンファーレ]**
  - 前半は `for` 文を使って音程を高速に上げながら「もぐもぐラッシュ」を表現し、後半は最高音まで一気に駆け上がるグリッサンドファンファーレを鳴らすという、最も豪華で楽しげな専用サウンド

## 3. このコード全体の設計の優れた点

- **これまでの「目・表情コード」との完璧な連携:**
  - これまでの `eye_animation.cpp` などで使われていた `EmotionType`（感情の定義）をそのまま共通の合図として受け取ることが可能なため、**「目が笑うと同時に、声も嬉しそうになる」というシンクロ挙動**が非常に簡単に実現可能
- **FreeRTOS等への配慮（テスト軽量版）:**
  - 複雑なタイマー割り込みや重いライブラリを使わず、Arduino標準の `delay()` や `delayMicroseconds()` をベースに矩形波を計算しているため、プログラム全体が非常に軽快で、移植やテストがしやすい設計になっている
