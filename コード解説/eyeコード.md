# eyeコード解説

```text
【ファイル構成】
eye_animation.h  : 定数・型定義・公開API宣言
eye_params.h/cpp : 感情パラメータテーブル（表情調整はここ）
eye_draw.h/cpp   : 目・鼻・口・アイテムの描画関数
eye_state.h/cpp  : まばたき・視線・感情遷移の状態管理
eye_animation.cpp: eyeUpdate / eyeUpdateRTC（このファイル）
```

## eye_amimation.cpp

- **かわいいキャラクターの「顔（目・鼻・口・エフェクト）」をアニメーション描画するためのメイン制御ロジック**

- 15fps（1秒間に15回）程度のループで呼び出されることを想定しており、現在の感情（`EmotionType`）や時間経過（`millis()`）に応じて顔の表情を動的に書き換える

### 1. 処理の全体フロー（レイヤー構造）

- `eyeUpdate` 関数の中では、画面がごちゃ混ぜにならないよう、まるでイラストソフトのように**レイヤー（階層）を意識して順番に描画**を行う
- 古い描画をクリアした後、以下の順にパーツを重ねていく

```text
[ 背景・バッファクリア ] (dsp.clearDisplay)
       ↓
【レイヤー1】 目（左右）の描画
       ↓
【レイヤー2】 特殊演出（エサやり時のハートなど）
       ↓
【レイヤー3】 感情エフェクト（涙 or ほっぺの照れ線）
       ↓
【レイヤー4】 顔パーツ（鼻・口）
       ↓
【レイヤー5】 小物アイテム（食べ物、タオル、Zzzなど）
       ↓
[ 画面転送 ] (dsp.display)
```

### 2. 各セクションの詳細解説

### ① 初期化（`eyeInit`）

```cpp
void eyeInit(Adafruit_SSD1306 &dsp) {
    eyeStateInit(); 
    Serial.println(F("[EYE] eyeInit completed."));
}
```

- 起動時に一度だけ呼ばれる関数
  - 内部で `eyeStateInit()`（別ファイルで定義された、まばたきや視線管理の初期化処理）を呼び出している

### ② 時間管理とアニメーション（口パクなど）

```cpp
unsigned long now   = millis();
int           frame = (int)((now / 500) % 2);
```

- Arduinoの起動からのミリ秒（`millis()`）を取得し、**500ミリ秒（0.5秒）ごとに `0` と `1` が交互に切り替わる `frame` 変数**を作成
- これにより、「もぐもぐ」という口パクや、ハートのフワフワした動きといった2コマのアニメーションを実現

### ③ 状態の計算（`eyeStateUpdate`）

```cpp
eyeStateUpdate(emotion, now, blinkLid, lidCut, gazeX, gazeY, gazeScale);

```

- 現在の感情や時刻をもとに、「今どれくらい目を閉じているか（まばたき）」「視線はどこを向いているか」を計算し、変数（`blinkLid` など）に結果を受け取る

### ④ 目の描画（レイヤー1）

```cpp
drawOneEye(dsp, EYE_L_CENTER_X, ..., true, emotion); // 左目
drawOneEye(dsp, EYE_R_CENTER_X, ..., false, emotion); // 右目
```

- 計算された視線位置（`gazeX`, `gazeY`）や、感情パラメータを元に左右の目を個別に描画する

### ⑤ 感情エフェクト：涙 or ほっぺ（レイヤー3）

```cpp
if (g_showTear) {
    // 涙の描画（目の下辺から物理シミュレーションで落ちる位置を計算）
    int tearBaseY = EYE_CENTER_Y + EYE_RADIUS_X; 
    int ty = tearBaseY + (int)g_tearOffset;
    dsp.fillCircle(EYE_L_CENTER_X - 8, ty, 1, WHITE);
    ...
} else if (emotion == EMOTION_HAPPY || ...) {
    // ほっぺ（照れ線）の描画
    drawCheek(...);
}
```

- `g_showTear` が `true`（悲しい時など）は、外側のシミュレーション処理から渡された位置（`g_tearOffset`）を使用し、ポロポロと落ちる涙を小さな円（`fillCircle`）で描画
- それ以外の通常〜嬉しい感情の時は、「ほっぺ（照れ線）」を描画

### ⑥ 小物アイテムの描画（レイヤー5）

```cpp
switch (emotion) {
    case EMOTION_EATING:  drawFoodItem(dsp, frame);              break;
    case EMOTION_SNACK:   drawSnackItem(dsp, frame);             break;
    case EMOTION_SLEEPING: drawSleepZzz(dsp, frame);             break;
    case EMOTION_BATH:    drawWaterLine(dsp, frame);
                          drawHeadTowel(dsp);                    break;
    ...
}
```

- キャラクターの現在の状態（ご飯中、おやつ中、睡眠中、お風呂中など）に合わせて、食べ物や「Zzz」の文字、頭の上のタオルといった**シチュエーション固有のアイテム**を上書き描画する

### ⑦ 画面の更新（チラつき防止）

```cpp
dsp.display();
```

- `clearDisplay()` から始まった一連の描画コマンドは、メモリ上の「バックバッファ」に対して行われている
- 最後に `dsp.display()` を呼ぶことで、完成した1枚の絵を一瞬でOLED画面に転送。これにより、**描画の途中経過が見えて画面がチラつくのを防いでいる**

### 3. このコード全体の設計の優れた点

- **密結合の回避（役割分担）:**
  - このファイル自身は「どの順番で何を呼ぶか」という司令塔（メインループ）に徹している
  - 「具体的な形の描き方」は `eye_draw.cpp` に、「まばたきのタイミングの計算」は `eye_state.cpp` に任せる設計（カプセル化）になっていて、綺麗に整理
- **拡張性の高さ:**
  - 新しい感情やアイテムを追加したい場合は、最後の `switch (emotion)`文や条件分岐に新しい関数を追加するだけで対応可能に設計

## eye_paeams.cpp

- キャラクターの表情（感情）ごとの数値を一元管理している「設定データシート（パラメータテーブル）」の役割を持つコード
- `eye_animation.cpp`（司令塔）や `eye_draw.cpp`（職人）は、このファイルに定義された数値を参照して、目の開き具合や黒目の動き幅を決定している
- 表情の微調整（デザインの変更）を行いたい場合は、基本的にこのファイルだけを書き換えることになるためシンプルな変更が可能

### 1. 構造体 `EyeParam` の中身（数値の意味）

- `switch (e)` 文で、それぞれの感情（`EmotionType`）に対して5つの数値をセットにして返している（`return { ... };`）
- それぞれの数値の意味は以下の通り

```cpp
// 構造体のデータの並び順
return { lidClose, lidCut, lashL, lashR, gazeScale };
```

1. **`lidClose` (まぶたの基本閉じ度):**
   - `0.0f`（全開）〜 `1.0f`（全閉）で指定
   - **例**: `0.4f` であればまぶたが上から40%ほど降りてきた「ジト目」のようなベース状態を作成

2. **`lidCut` (追加カット量):**
   - 現在は使われておらず将来の拡張用（予約）

3. **`lashL` / `lashR` (まつ毛ライン表示フラグ):**
   - コード内のコメントでは `lashL`, `lashR` となっており（ヘッダー等の定義）、ファイルの冒頭説明にある `tearLeft`, `tearRight` からアップデートされたもの
   - まつ毛や目のフチのラインを描画するかどうかの `true / false`（ON/OFF）を切り替え

4. **`gazeScale` (黒目のキョロキョロ範囲):**
   - 目線（キョロキョロ動く範囲）の倍率
   - `1.0f` が通常で、数値を小さくするほど黒目があまり動かなくなる
   - 眠い時や集中している時は動きを鈍くする（数値を小さくする）演出に使用

### 2. 感情ごとのパラメータ設定の意図（解説）

- キャラクターの心理状態が数値で巧みに表現

- **`EMOTION_NORMAL` (通常) / `EMOTION_HAPPY` (嬉しい)**
  - `{ 0.0f, 0.0f, true, true, 1.0f }`
  - まぶたはパッチリ全開（`0.0f`）で、視線も通常通りキョロキョロ動く（`1.0f`）

- **`EMOTION_ANGRY` (怒り) / `EMOTION_SAD` (悲しみ)**
  - 怒り：`{ 0.4f, ... }` / 悲しみ：`{ 0.35f, ... }`
  - まぶたをあらかじめ35〜40%ほど下げた状態（ジト目気味）にしている
  - `eye_draw.cpp` では、この下がったまぶたのラインを基準にして、釣り目（怒り）やタレ目（悲しみ）の黒いマスクを重ねることで表情を作成

- **`EMOTION_CONFUSED` (困惑)**
  - `{ 0.25f, 0.0f, false, false, 0.6f }`
  - まぶたを少し下げ、まつ毛フラグを `false` にして目をすっきり（あるいは点目に）させ、さらに視線の動き幅を `0.6f` に狭めることで「オドオドして目が泳ぐ・あるいは固まる」様子を表現している

- **`EMOTION_SLEEPY` (うとうと・眠気)**
  - `{ 0.45f, 0.0f, false, false, 0.3f }`
  - まぶたを半分近く閉じ（`0.45f`）、視線の動きをかなり小さく（`0.3f`）して、とろんとした眠そうな目を表現
  - **独自の処理:** `sleepyCnt`（うとうとする回数）が `0`（未設定）のとき、`random(3, 6)` を使って「3回〜5回うとうとさせる」というランダムな回数を内部で決める仕掛けが入っている

- **`EMOTION_SLEEPING` (睡眠)**
  - `{ 0.95f, 0.0f, false, false, 0.0f }`
  - まぶたの閉じ度が `0.95f`（ほぼ全閉）になり、黒目の動き幅も `0.0f`（完全に固定）になる
  - これにより、`eye_draw.cpp` 側で「完全に目を閉じている状態（睡眠の線）」が描画される

- **`EMOTION_EATING` (ご飯) / `EMOTION_SNACK` (おやつ)**
  - `{ 0.0f, 0.0f, true, true, 0.5f }`
  - 目はパッチリ開けるが、食べることに集中させるため、黒目のキョロキョロ範囲を半分（`0.5f`）に抑えている

### 3. このファイルに施された設計のメリット

- **デザイン調整のイージー化:**
  - 「怒ったときにもう少し目をカッと見開いてほしい」と思ったら、`EMOTION_ANGRY` の `0.4f` を `0.2f` に書き換えるだけで済む。描画ロジック（`eye_draw.cpp`）を一切触る必要がない
- **不要な処理の整理（リファクタリング）:**
  - 冒頭のコメントにある通り、以前はここで「時刻から感情を決める処理（`getCurrentEventEmotion`）」も行っていたが、それをメインプログラム（`robot_pet.ino` の `TaskEmotion`）に一任するように修正
  - これにより、「データ管理はこのファイル」「時間管理はメインファイル」という単一責任の原則（役割の明確化）がより徹底されている

## eye_draw.cpp

- 司令塔（`eye_animation.cpp`）から指示を受けて、**実際に有機ELディスプレイ（OLED）上にドット絵や図形を描画する「職人」の役割を持つコード**
- Adafruit_SSD1306ライブラリのグラフィック関数（点、線、円、四角形の塗りつぶしなど）を駆使して、キャラクターの豊かな表情や小物を表現している

### 1. 最も重要な関数：`drawOneEye`（目の描画）

- このファイルの核となる関数
  - 感情（`emotion`）や「まばたき（`lidCut`）」の計算結果に応じて、1つの目を組み立てる

#### A. 白目と黒目の組み合わせ

- 基本的には、まず大きな白い円（`fillCircle`）で**白目**を描き、その上に一回り小さな黒い円で黒目（瞳）を描くことで「大きな黒目」スタイルを表現

- **視線のキョロキョロ表現:**
  - 引数 `gx`, `gy`（視線ベクトル）を使用し、黒目の中心座標をずらしている
  - 白目からはみ出さないように `constrain` 関数でブレーキをかける安全設計

#### B. 感情ごとの「白目カット」テクニック

- キャラクターの「ジト目」や「怒り」「悲しみ」を表現するために、**白い円を描いた後、黒い四角（`fillRect`）や三角形（`fillTriangle`）を上から重ねて白目を削る**というマスク処理を使用

- **`EMOTION_ANGRY` (怒り):** 目の上側を、中央に向かって釣り上がるように黒い三角形で削る
- **`EMOTION_SAD` (悲しみ):** 逆に、中央に向かってタレ目（ハの字）になるように削る

#### C. まぶたの表現（シャッター方式）

- まばたきや、困惑（`CONFUSED`）のときは、目の上部から下方向へ向かって黒い長方形（`fillRect`）を上書きすることで、シャッターが閉まるように目を閉じさせている
- さらに、そのまぶたの境界線に白い横線（`drawFastHLine`）を1本引くことで、綺麗な「まつ毛（まぶたのフチ）」を表現

### 2. 表情を豊かにするパーツ描画

### ① ほっぺ（`drawCheek`）

- 目のすぐ下に、斜め線（`drawLine`）を左右に3本ずつ描くことで、アニメにあるような「照れ線（///）」を描画
- 画面の左側か右側かを判定して自動で描画位置の左右オフセット（`-8` または `8`）を調整

### ② 口（`drawMouth`）

- 感情に合わせたドットの配置が細かく定義されている

- **`EMOTION_HAPPY` など:** 1つずつドット（`drawPixel`）を打って、可愛い「**ω**（W型）」の口を描く
- **`EMOTION_EATING` / `SNACK` (もぐもぐ):** `frame` 変数を使用し、口を開けた四角（`drawRect`）と、閉じた横線（`drawFastHLine`）を交互に切り替えることで、実際に食べているようなアニメーションを実現

### 3. シチュエーションを彩る「小物アイテム」

- キャラクターの状態に応じて画面の隅に描画される、アイディア満載のドット絵ロジック

| 関数名 | 描画されるもの | アニメーションの工夫 |
| --- | --- | --- |
| `drawFoodItem` | ご飯茶碗 | `frame` によって、だんだん減っていく演出 |
| `drawSnackItem` | クッキー | `frame` が進むにつれて、黒い四角で右側を削ることで**「だんだんかじられて減っていく」**演出 |
| `drawSleepZzz` | Zzz...（文字） | 大文字の「Z」と小文字の「z」の位置を交互にずらして、寝息が浮き上がっているように見せる |
| `drawWaterLine` | お風呂の波線 | 画面最下部に、1ドットずらした横線を交互に描いて波のパタパタ感を表現 |
| `drawFloatingHearts` | ハートマーク | ラムダ式（関数内のミニ関数）でハートの形を定義し、時間経過（`frame`）とともにY座標をマイナス（上方向）へ移動させて**フワフワ浮き上がる演出** |

## `eye_state.cpp`

- キャラクターの「命の息吹」とも言える**まばたきのタイミング、視線のキョロキョロとした動き、感情が切り替わる瞬間の滑らかな変化をリアルタイムに計算する「脳（状態管理エンジン）」の役割を持つコード**
- このファイル自身は画面に何かを描画することはせず、あくまで内部で「今、目はどれくらい開いているか？」「視線はどこを向いているか？」の数値を毎フレーム計算し、その結果を先ほどの描画レイヤーへ渡す役割に徹している

### 1. 感情遷移の「線形補間（ラープ）」

- 感情が切り替わった瞬間、目がパッと一瞬で切り替わるとロボット感が強く不自然に見えてしまうため、このコードでは `lerpF`（線形補間関数）を使用し、300ミリ秒かけてジワリと次の表情へ変化させる（**モーフィング**）処理を行っている
- `target`（次の表情パラメータ）に向かって、`morphRate`（時間経過の割合）を使い、現在のまぶたの閉じ度（`s_curLidClose`）や視線のスケール（`s_curGazeScale`）を少しずつ近づけていく。これにより、生物らしいなめらかな表情の移り変わりを実現している

### 2. 賢いまばたきロジック（有限状態機械）

- 目がパチパチする「まばたき」は、ランダムな時間間隔（`random(BLINK_INTERVAL_MIN, BLINK_INTERVAL_MAX)`）で発生するよう制御している。まばたき状態は、内部で細かくフラグ管理されてしている

- **通常のまばたき:**
  - 「目を閉じている最中（`s_blinkClosing`）」→「完全に閉じた」→「目を開けている最中（`s_blinkOpening`）」というステップを高速（スピード `0.22f`）で行う
- **「うとうと（`EMOTION_SLEEPY`）」時の特殊処理:**
  - 眠い時はまばたきのスピードがかなり遅く（`0.08f` または `0.03f`）なる
  - さらに、ランダムに決定された回数（`sleepyBlinkCount`）だけパチパチした後に、1200ミリ秒（1.2秒）の間、目を完全に閉じてキープする（`s_blinkHolding`）という「うとうとして寝落ちしかけている」リアルな挙動をプログラムしている

### 3. 自律的な視線（キョロキョロ）の計算

- キャラクターが生きているように見えるよう、一定時間ごとに視線の目標位置（`s_gazeTargX`, `s_gazeTargY`）をランダムに決定している

- 目標が決まると、`s_gazeMoving = true` となり、150ミリ秒かけてその方向へ黒目をスムーズに動かしている
- これにより、ただ一点を見つめるのではなく、時折フッと目線を動かす生き物らしさが生まれる

### 4. 特殊エフェクト：涙の物理シミュレーションとバグ修正

```cpp
// ── 涙オフセット更新 ──
if (s_emotion == EMOTION_SAD) {
    s_tearOffset = (s_tearOffset + 0.5f > 12.0f) ? 0.0f : s_tearOffset + 0.5f;
    g_tearOffset = s_tearOffset;
    g_showTear   = true;
} else {
    g_showTear = false;
}
```

- 悲しい感情（`EMOTION_SAD`）のときは、毎フレーム `s_tearOffset` を `0.5f` ずつ下にずらしていく
  - 12.0ピクセル分まで下に流れたら、再び `0.0f`（目のきわ）に戻すというループ処理を行うことで、**涙がポロポロと数滴こぼれ落ち続けるアニメーション**を作成
